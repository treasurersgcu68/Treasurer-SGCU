# คู่มือการใช้งาน Template “เลขรันเอกสาร”

คู่มือนี้จัดทำจากโครงสร้างชีทจริงในไฟล์ `[SGCUXX-10 TEMPLATE] เลขรันเอกสาร.xlsx` สำหรับใช้เป็นคำแนะนำในการตั้งค่าและใช้งาน Template เลขรันเอกสารของฝ่ายเหรัญญิก ตั้งแต่การทำสำเนาไฟล์ การตั้งค่าปีการศึกษา การเชื่อม Project Status ไปจนถึงการกรอกข้อมูลในแต่ละชีท

## 1. ภาพรวมของ Template

Template นี้ใช้สำหรับบันทึกและสร้างเลขรันเอกสารของฝ่ายเหรัญญิกและฝ่ายที่เกี่ยวข้อง โดยอาศัยข้อมูล 2 ส่วนหลัก

1. ข้อมูลที่ผู้ใช้กรอกเองในชีทเอกสาร
2. ข้อมูลที่ระบบดึงจาก Project Status ผ่านชีทหลังบ้าน `SETTING`

เลขรันจะถูกสร้างจาก “ประเภทเอกสาร” “เลขฝ่าย” “ลำดับแถว” และ “ปีการศึกษา” ตามสูตรที่กำหนดไว้ในแต่ละชีท ผู้ใช้จึงควรกรอกเฉพาะช่องที่กำหนด และไม่ควรแก้ไขสูตรหรือหัวคอลัมน์เดิม

## 2. วิธีเริ่มต้นใช้งาน

### 2.1 ทำสำเนาไฟล์ Template

1. เปิดลิงก์ Google Sheets Template “เลขรันเอกสาร”
2. ไปที่เมนู `File > Make a copy` หรือ `ไฟล์ > ทำสำเนา`
3. ตั้งชื่อไฟล์ตามปีการศึกษาที่ใช้งาน เช่น `SGCU68-10 เลขรันเอกสาร`
4. เลือกโฟลเดอร์จัดเก็บที่ผู้รับผิดชอบในฝ่ายสามารถเข้าถึงได้
5. ตรวจสอบสิทธิ์การเข้าถึงไฟล์ โดยให้เฉพาะผู้เกี่ยวข้องมีสิทธิ์แก้ไข

### 2.2 ตั้งค่าหน้าแรก

ให้เข้าไปที่ชีท `เลขรันเอกสาร` ซึ่งเป็นชีทหน้าหลักของไฟล์

| ตำแหน่ง | รายการ | วิธีใช้งาน |
|---|---|---|
| `A1` | ปีการศึกษา | ช่องหัวข้อกำกับข้อมูลปีการศึกษา |
| `B1` | ปีการศึกษา | ให้กรอกปีการศึกษาที่ใช้งาน เช่น `2568` |
| `A2` | Project Status | ช่องหัวข้อกำกับลิงก์ Project Status |
| `B2` | Project Status | ให้แนบลิงก์ Google Sheets หรือ URL ที่ใช้กับ `IMPORTRANGE` ของ Project Status ปีนั้น |

ลิงก์ Project Status ใน `B2` เป็นแหล่งข้อมูลที่ชีทหลังบ้าน `SETTING` ใช้ดึงข้อมูลโครงการ เช่น รหัสโครงการ ชื่อโครงการ หน่วยงาน ฝ่าย สถานะปิดโครงการ และสถานะที่เกี่ยวข้องกับเอกสารยืมรองจ่าย/ฎีกา หากลิงก์ไม่ถูกต้อง ชีทเอกสารพิเศษจะไม่สามารถเติมข้อมูลอัตโนมัติได้

### 2.3 ตรวจรายการประเภทเอกสาร

ในชีท `เลขรันเอกสาร` แถวที่ 4 เป็นต้นไปเป็นตารางกำหนดประเภทเอกสารและเลขฝ่าย

| คอลัมน์ | หัวข้อ | วิธีใช้งาน |
|---|---|---|
| `A` | ประเภทเอกสาร | รายชื่อประเภทเอกสาร เช่น เหรัญญิก, เอกสารยืมรองจ่าย, ฎีกา |
| `B` | เลขฝ่าย | เลขหมวดเอกสารหลัก เช่น `10.0`, `10.01.01`, `10.02` |
| `C` | เลขฝ่าย | สูตร/ผลลัพธ์ตัวอย่างรูปแบบเลขฝ่ายร่วมกับปีการศึกษา |

ตัวอย่างรายการที่พบในไฟล์

| ประเภทเอกสาร | เลขฝ่าย | ตัวอย่างรูปแบบเลข |
|---|---:|---|
| เหรัญญิก | `10.0` | `10-XX/ปีการศึกษา` |
| เอกสารยืมรองจ่าย | `10.01.01` | `10.01.01-XX/ปีการศึกษา` |
| เอกสารเปลี่ยนเพดานงบ | `10.01.02` | `10.01.02-XX/ปีการศึกษา` |
| เอกสารเรียกเงินยืมรองจ่ายคืน | `10.01.03` | `10.01.03-XX/ปีการศึกษา` |
| บริหารและพัฒนางบประมาณ | `10.02` | `10.02-XX/ปีการศึกษา` |
| หาทุนและสิทธิประโยชน์ | `10.03` | `10.03-XX/ปีการศึกษา` |
| กายภาพและพัสดุ | `10.04` | `10.04-XX/ปีการศึกษา` |

## 3. โครงสร้างชีททั้งหมด

จากไฟล์ที่ตรวจพบ มีชีททั้งหมด 10 ชีท แบ่งได้เป็น 3 กลุ่มการใช้งาน

### 3.1 ชีทหน้าหลัก

| ชื่อชีท | ใช้สำหรับ |
|---|---|
| `เลขรันเอกสาร` | ตั้งค่าปีการศึกษา ลิงก์ Project Status และตารางประเภทเอกสาร/เลขฝ่าย |

### 3.2 เอกสารทั่วไป

เอกสารทั่วไปเป็นชีทที่มีโครงสร้างเหมือนกัน ใช้บันทึกเอกสารรายฝ่าย โดย 1 แถวหมายถึงเอกสาร 1 ฉบับ

| ชื่อชีท | ใช้สำหรับ |
|---|---|
| `เหรัญญิก` | บันทึกเลขรันเอกสารทั่วไปของฝ่ายเหรัญญิก |
| `บริหารและพัฒนางบประมาณ` | บันทึกเลขรันเอกสารของฝ่ายบริหารและพัฒนางบประมาณ |
| `หาทุนและสิทธิประโยชน์` | บันทึกเลขรันเอกสารของฝ่ายหาทุนและสิทธิประโยชน์ |
| `กายภาพและพัสดุ` | บันทึกเลขรันเอกสารของฝ่ายกายภาพและพัสดุ |

### 3.3 เอกสารพิเศษ

เอกสารพิเศษเป็นชีทที่มีสูตรและการดึงข้อมูลจาก Project Status มากกว่าเอกสารทั่วไป

| ชื่อชีท | ใช้สำหรับ |
|---|---|
| `เอกสารยืมรองจ่าย` | ออกเลขเอกสารที่เกี่ยวข้องกับการยืมเงินรองจ่าย |
| `เอกสารขอเปลี่ยนเพดานงบ` | ออกเลขเอกสารสำหรับการลด/เพิ่มเพดานงบประมาณระหว่างโครงการ |
| `ฎีกา` | ออกเลขเอกสารเรียกเงินยืมรองจ่ายคืน/ฎีกา โดยแยกเลขตามประเภทองค์กรและสถานะปิดโครงการ |

### 3.4 ชีทระบบหลังบ้าน

| ชื่อชีท | สถานะ | ใช้สำหรับ |
|---|---|---|
| `TEMPLATE` | แสดงอยู่ | ต้นแบบสำหรับคัดลอกไปสร้างชีทเอกสารทั่วไปเพิ่มเติม |
| `SETTING` | ซ่อนอยู่ | ดึงและแปลงข้อมูลจาก Project Status ด้วยสูตร เช่น `IMPORTRANGE`, `FILTER`, `QUERY`, `ARRAYFORMULA` |

## 4. วิธีใช้งานเอกสารทั่วไป

ชีทเอกสารทั่วไป ได้แก่ `เหรัญญิก`, `บริหารและพัฒนางบประมาณ`, `หาทุนและสิทธิประโยชน์` และ `กายภาพและพัสดุ` ใช้รูปแบบเดียวกันทั้งหมด

### 4.1 โครงสร้างคอลัมน์

| คอลัมน์ | หัวข้อ | ประเภทข้อมูล | วิธีใช้งาน |
|---|---|---|---|
| `A` | ลำดับ | สูตร/อัตโนมัติ | สร้างลำดับจากจำนวนรายการในคอลัมน์ `C` ห้ามแก้ไข |
| `B` | เลขรัน | สูตร/อัตโนมัติ | สร้างเลขรันเมื่อกรอก `รายการ` ในคอลัมน์ `C` ห้ามแก้ไข |
| `C` | รายการ | ผู้ใช้กรอกเอง | กรอกชื่อเอกสารหรือเรื่องที่ต้องออกเลข |
| `D` | ผู้รับผิดชอบ | ผู้ใช้กรอกเอง | กรอกชื่อผู้รับผิดชอบเอกสาร |
| `E` | สถานะ | ผู้ใช้เลือก/กรอกเอง | เลือกจากรายการ: `ยกเลิก`, `ร่าง`, `รอลงนาม`, `ส่งแล้ว` |
| `F` | หมายเหตุ | ผู้ใช้กรอกเอง | กรอกข้อมูลเพิ่มเติม เช่น เหตุผลที่ยกเลิก วันที่ส่ง หรือรายละเอียดประกอบ |

### 4.2 การทำงานของสูตร

สูตรในชีทเอกสารทั่วไปมี 2 จุดหลัก

| ตำแหน่ง | การทำงาน |
|---|---|
| `A4` เป็นต้นไป | ใช้ `SEQUENCE(COUNTA(C4:C1000),1)` เพื่อสร้างลำดับตามจำนวนรายการที่กรอกในคอลัมน์ `C` |
| `B4` เป็นต้นไป | ถ้าคอลัมน์ `C` มีข้อมูล จะสร้างเลขรันรูปแบบ `อบจ. [เลขฝ่าย]-[เลขลำดับ 3 หลัก]/[ปีการศึกษา]` |

ตัวอย่างรูปแบบเลขรัน เช่น ถ้าอยู่ในชีท `เหรัญญิก` และตั้งปีการศึกษาเป็น `2568` รายการแถวแรกจะได้เลขลักษณะ `อบจ. 10.0-001/2568`

### 4.3 ตัวอย่างการกรอก 1 แถว

| ลำดับ | เลขรัน | รายการ | ผู้รับผิดชอบ | สถานะ | หมายเหตุ |
|---:|---|---|---|---|---|
| อัตโนมัติ | อัตโนมัติ | ขออนุมัติจัดซื้อวัสดุสำนักงาน | ชื่อผู้รับผิดชอบ | ร่าง | รอแนบใบเสนอราคา |

ให้ผู้ใช้กรอกเฉพาะ `รายการ`, `ผู้รับผิดชอบ`, `สถานะ` และ `หมายเหตุ` ส่วน `ลำดับ` และ `เลขรัน` ให้ปล่อยเป็นสูตร

### 4.4 ข้อควรระวังสำหรับเอกสารทั่วไป

- ห้ามแก้ไขสูตรในคอลัมน์ `A` และ `B`
- ห้ามเปลี่ยนหัวคอลัมน์แถวที่ 3
- ห้ามลบแถวสูตรเริ่มต้นที่แถว 4
- หากต้องการยกเลิกเอกสาร ให้เปลี่ยนสถานะเป็น `ยกเลิก` และใส่เหตุผลใน `หมายเหตุ` ไม่ควรลบแถวนั้นทิ้ง
- ถ้าต้องการเพิ่มเอกสารใหม่ ให้กรอกต่อในแถวว่างถัดไป
- หากเลขรันไม่ขึ้น ให้ตรวจว่ากรอก `รายการ` ในคอลัมน์ `C` แล้วหรือไม่ และตรวจว่าชีท `เลขรันเอกสาร` มีปีการศึกษาใน `B1`

## 5. วิธีใช้งานชีท `เอกสารยืมรองจ่าย`

### 5.1 ใช้ในกรณีใด

ใช้สำหรับออกเลขเอกสารที่เกี่ยวข้องกับการยืมเงินรองจ่ายของโครงการ โดยชีทนี้ผูกกับ Project Status เพื่อดึงข้อมูลชื่อโครงการ ฝ่าย และชื่อหน่วยงานจากรหัสโครงการ

### 5.2 โครงสร้างคอลัมน์

| คอลัมน์ | หัวข้อ | ประเภทข้อมูล | วิธีใช้งาน |
|---|---|---|---|
| `A` | ลำดับ | สูตร/อัตโนมัติ | สร้างลำดับจากจำนวนรหัสโครงการในคอลัมน์ `D` ห้ามแก้ไข |
| `B` | เลขรัน | สูตร/อัตโนมัติ | สร้างเลขรันเมื่อกรอกข้อมูลครบตามเงื่อนไข ห้ามแก้ไข |
| `C` | STATUS | สูตร/อัตโนมัติ/มีรายการสถานะ | ดึงสถานะที่สัมพันธ์กับเลขรันจาก `SETTING` หรือใช้รายการสถานะที่กำหนดไว้ ควรตรวจสอบสูตรในชีทจริงอีกครั้งก่อนแก้ไข |
| `D` | รหัสโครงการ | ผู้ใช้เลือก/กรอกเอง | เลือกรหัสโครงการจากข้อมูล Project Status ผ่าน `SETTING!L2:L502` |
| `E` | รายการ | สูตร/อัตโนมัติ | ดึงชื่อโครงการจาก Project Status ผ่าน `SETTING` ห้ามแก้ไข |
| `F` | โครงการอยู่ภายใต้ฝ่าย | สูตร/อัตโนมัติ | ดึงฝ่ายจาก Project Status ห้ามแก้ไข |
| `G` | ชื่อหน่วยงาน | สูตร/อัตโนมัติ | ดึงชื่อหน่วยงานจาก Project Status ห้ามแก้ไข |
| `H` | ผู้รับผิดชอบ | ผู้ใช้กรอกเอง | กรอกชื่อผู้รับผิดชอบเอกสาร |
| `I` | สถานะ | ผู้ใช้เลือก/กรอกเอง | เลือกจากรายการ: `ยกเลิก`, `ร่าง`, `รอลงนาม`, `ส่งแล้ว` |
| `J` | หมายเหตุ | ผู้ใช้กรอกเอง | กรอกรายละเอียดเพิ่มเติม |

### 5.3 เงื่อนไขการออกเลขรัน

สูตรในคอลัมน์ `B` จะสร้างเลขรันเมื่อข้อมูลครบ 3 ส่วน ได้แก่

1. `D` รหัสโครงการ
2. `H` ผู้รับผิดชอบ
3. `I` สถานะ

ถ้าทั้ง 3 ช่องว่างทั้งหมด เลขรันจะว่าง หากกรอกบางช่องแต่ยังไม่ครบ สูตรจะแสดงข้อความ `กรอกข้อมูลไม่ครบถ้วน`

### 5.4 สถานะในคอลัมน์ `STATUS`

คอลัมน์ `C` ผูกกับข้อมูลใน `SETTING` โดยสูตรค้นหาเลขรันจาก `SETTING!U2:U502` แล้วคืนค่าสถานะจาก `SETTING!T2:T502` สถานะที่พบในรายการตรวจสอบของชีทนี้ ได้แก่

- ยังไม่เริ่มดำเนินการ
- องค์กรกำลังดำเนินการ
- ผู้ช่วยตรวจสอบ
- เลขาตรวจสอบ
- รอนายกลงนาม
- ส่งกิจการนิสิตแล้ว รออนุมัติ
- รอแก้ไข
- สำรองจ่ายก่อน
- เหรัญญิกรับเงินจากกิจการนิสิต
- โครงการรับเงินแล้ว
- ยกเลิก
- ไม่อนุมัติ / อนุมัติไม่ทันวันจัดกิจกรรม

เนื่องจากสถานะนี้สัมพันธ์กับ Project Status และสูตรหลังบ้าน ควรตรวจสอบสูตรในชีทจริงอีกครั้งก่อนแก้ไข logic สถานะ

### 5.5 ตัวอย่างการกรอก

| รหัสโครงการ | ผู้รับผิดชอบ | สถานะ | หมายเหตุ |
|---|---|---|---|
| เลือกจาก dropdown | ชื่อผู้รับผิดชอบ | ร่าง | รอเอกสารจากหน่วยงาน |

หลังกรอก `รหัสโครงการ`, `ผู้รับผิดชอบ` และ `สถานะ` ครบ ชีทจะสร้าง `ลำดับ` และ `เลขรัน` ให้โดยอัตโนมัติ พร้อมดึง `รายการ`, `โครงการอยู่ภายใต้ฝ่าย` และ `ชื่อหน่วยงาน` จาก Project Status

### 5.6 ข้อควรระวัง

- ห้ามแก้ไขคอลัมน์ `A`, `B`, `C`, `E`, `F`, `G`
- หากเลือก `รหัสโครงการ` แล้วชื่อโครงการไม่ขึ้น ให้ตรวจลิงก์ Project Status ในชีท `เลขรันเอกสาร!B2`
- หากขึ้น `กรอกข้อมูลไม่ครบถ้วน` ให้ตรวจ `D`, `H`, `I`
- หากขึ้น `ไม่พบข้อมูล` ให้ตรวจว่ารหัสโครงการมีอยู่ใน Project Status และข้อมูลถูกนำเข้าใน `SETTING` แล้วหรือไม่

## 6. วิธีใช้งานชีท `เอกสารขอเปลี่ยนเพดานงบ`

### 6.1 ใช้ในกรณีใด

ใช้สำหรับออกเลขเอกสารเมื่อมีการเปลี่ยนเพดานงบประมาณระหว่างโครงการ เช่น ลดงบประมาณจากบางโครงการ แล้วเพิ่มงบประมาณให้โครงการอื่น

### 6.2 โครงสร้างคอลัมน์

| คอลัมน์ | หัวข้อ | ประเภทข้อมูล | วิธีใช้งาน |
|---|---|---|---|
| `A` | ลำดับ | สูตร/อัตโนมัติ | สร้างลำดับตามจำนวนเลขรันในคอลัมน์ `B` ห้ามแก้ไข |
| `B` | เลขรัน | สูตร/อัตโนมัติ | สร้างเลขรันเมื่อกรอกโครงการอย่างน้อย 1 ช่อง และกรอกสถานะใน `M` |
| `C:G` | โครงการลดงบประมาณ 1-5 | ผู้ใช้เลือก/กรอกเอง | เลือกรหัสโครงการที่ต้องการลดงบประมาณ |
| `H:L` | โครงการเพิ่มงบประมาณ 1-5 | ผู้ใช้เลือก/กรอกเอง | เลือกรหัสโครงการที่ต้องการเพิ่มงบประมาณ |
| `M` | สถานะ | ผู้ใช้เลือก/กรอกเอง | เลือกจากรายการ: `ยกเลิก`, `ร่าง`, `รอลงนาม`, `ส่งแล้ว` |
| `N` | หมายเหตุ | ผู้ใช้กรอกเอง | กรอกรายละเอียดเพิ่มเติม |
| `O:T` | คอลัมน์หลังบ้านที่ซ่อนอยู่ | สูตร/ระบบ | ใช้รวม/แปลงข้อมูลโครงการจากช่วง `C:L` ห้ามแก้ไข |

### 6.3 เงื่อนไขการออกเลขรัน

สูตรในคอลัมน์ `B` จะสร้างเลขรันเมื่อครบ 2 เงื่อนไข

1. มีรหัสโครงการอย่างน้อย 1 ช่องในช่วง `C:L`
2. มีสถานะในคอลัมน์ `M`

หากยังไม่กรอกสถานะ เลขรันจะยังไม่แสดง แม้จะเลือกรหัสโครงการแล้ว

### 6.4 ตัวอย่างการใช้งาน

| โครงการลดงบประมาณ 1 | โครงการเพิ่มงบประมาณ 1 | สถานะ | หมายเหตุ |
|---|---|---|---|
| รหัสโครงการที่ลดงบ | รหัสโครงการที่เพิ่มงบ | ร่าง | รออนุมัติจากผู้เกี่ยวข้อง |

### 6.5 ข้อควรระวัง

- ช่วง `C:L` มี dropdown จาก `SETTING!B2:B502` ควรเลือกรหัสโครงการจากรายการ ไม่ควรพิมพ์เองหากไม่จำเป็น
- ห้ามแก้ไขคอลัมน์ `A`, `B` และคอลัมน์ซ่อน `O:T`
- ห้ามลบหรือย้ายคอลัมน์ เพราะสูตรหลังบ้านใช้ตำแหน่งคอลัมน์ในการรวมข้อมูล
- หากเลขรันไม่ขึ้น ให้ตรวจว่ามีรหัสโครงการใน `C:L` และมีสถานะใน `M`

## 7. วิธีใช้งานชีท `ฎีกา`

### 7.1 ใช้ในกรณีใด

ชีท `ฎีกา` ใช้สำหรับออกเลขเอกสารเรียกเงินยืมรองจ่ายคืน/ฎีกา โดยดึงข้อมูลโครงการจาก Project Status และสร้างเลขรันตามประเภทองค์กรหรือสถานะ `ไม่ส่งปิดโครงการ`

### 7.2 โครงสร้างคอลัมน์

| คอลัมน์ | หัวข้อ | ประเภทข้อมูล | วิธีใช้งาน |
|---|---|---|---|
| `A` | ลำดับ | สูตร/อัตโนมัติ | สร้างลำดับตามจำนวนรหัสโครงการในคอลัมน์ `D` ห้ามแก้ไข |
| `B` | เลขรันเอกสาร | สูตร/อัตโนมัติ | สร้างเลขรันตามประเภทองค์กรและสถานะปิดโครงการ ห้ามแก้ไข |
| `C` | STATUS | สูตร/อัตโนมัติ/มีรายการสถานะ | ดึงสถานะจาก `SETTING` โดยอ้างอิงเลขรันเอกสาร หรือเลือกสถานะตามรายการที่กำหนดไว้ ควรตรวจสอบสูตรในชีทจริงอีกครั้งก่อนแก้ไข |
| `D` | รหัสโครงการ | ผู้ใช้เลือก/กรอกเอง | เลือกรหัสโครงการจาก `SETTING!B2:B502` |
| `E` | ชื่อโครงการ | สูตร/อัตโนมัติ | ดึงชื่อโครงการจาก Project Status |
| `F` | ชื่อหน่วยงาน | สูตร/อัตโนมัติ | ดึงชื่อหน่วยงานจาก Project Status |
| `G` | โครงการอยู่ภายใต้ฝ่าย | สูตร/อัตโนมัติ | ดึงฝ่ายจาก Project Status |
| `H` | สถานะปิดโครงการ | สูตร/อัตโนมัติ/มีรายการสถานะ | ดึงสถานะปิดโครงการจาก Project Status หรือเลือกจากรายการสถานะที่กำหนด |
| `I:S` | คอลัมน์หลังบ้านที่ซ่อนอยู่ | สูตร/ระบบ | ใช้แยกประเภทองค์กรและนับลำดับเลขรันตามหมวด ห้ามแก้ไข |

### 7.3 การทำงานของเลขรัน

สูตรในคอลัมน์ `B` ทำงานโดยตรวจข้อมูลหลัก 2 ส่วน

1. `D` รหัสโครงการ
2. `G` โครงการอยู่ภายใต้ฝ่าย

หากมีข้อมูลครบ สูตรจะสร้างเลขรันตามเงื่อนไขต่อไปนี้

| เงื่อนไข | รูปแบบเลขรัน |
|---|---|
| สถานะปิดโครงการใน `H` เป็น `ไม่ส่งปิดโครงการ` | `อบจ. [เลขฝ่าย]-[เลขลำดับ 3 หลัก]/[ปีการศึกษา]` |
| เป็นองค์กร/ชมรมทั่วไป | `[รหัสกลุ่มองค์กร][เลขท้ายปี]/[เลขลำดับ 3 หลัก]` เช่น `SGCU68/001` |

รหัสกลุ่มองค์กรที่พบในสูตร ได้แก่

| ข้อมูลในคอลัมน์ `G` | รหัสที่ใช้ในเลขรัน |
|---|---|
| องค์การบริหารสโมสรนิสิต | `SGCU` |
| องค์การบริหารสโมสรนิสิต, สภานิสิต | `SGCU` |
| สภานิสิต | `SCCU` |
| ชมรมฝ่ายศิลปะและวัฒนธรรม | `ART` |
| ชมรมฝ่ายวิชาการ | `VCK` |
| ชมรมฝ่ายกีฬา | `SPT` |
| ชมรมฝ่ายพัฒนาสังคมและบำเพ็ญประโยชน์ | `PHT` |

### 7.4 สถานะที่เกี่ยวข้อง

รายการสถานะในคอลัมน์ `H` ที่พบในชีท ได้แก่

- รอส่งเอกสารปิดโครง
- รอผู้ช่วยตรวจสอบ
- ผู้ช่วยเหรัญญิกตรวจสอบ
- ผ่านผู้ช่วยเหรัญญิก
- เหรัญญิกตรวจสอบ
- ผ่านเหรัญญิก
- ส่งให้ชมรมกลับเพื่อแก้ไข
- ชมรมรับเอกสารแก้ไข
- รอส่งกิจการนิสิต
- ส่งกิจการนิสิตเรียบร้อย
- ไม่ส่งปิดโครงการ

รายการสถานะในคอลัมน์ `C` ที่พบในชีท ได้แก่

- เหรัญญิกรับเงินจากกิจการนิสิต
- โครงการรับเงินแล้ว
- โครงการคืนเงินแล้ว
- รอเสนอเอกสารอนุมัติ
- รอปิดโครงการ

### 7.5 ตัวอย่างการใช้งานจริง

ตัวอย่างที่ 1: โครงการสังกัดองค์การบริหารสโมสรนิสิต

| รหัสโครงการ | ชื่อโครงการ | โครงการอยู่ภายใต้ฝ่าย | สถานะปิดโครงการ | เลขรัน |
|---|---|---|---|---|
| เลือกจาก dropdown | ดึงอัตโนมัติ | องค์การบริหารสโมสรนิสิต | รอส่งเอกสารปิดโครง | `SGCU[เลขท้ายปี]/[ลำดับ]` |

ตัวอย่างที่ 2: โครงการที่ไม่ส่งปิดโครงการ

| รหัสโครงการ | สถานะปิดโครงการ | เลขรัน |
|---|---|---|
| เลือกจาก dropdown | ไม่ส่งปิดโครงการ | `อบจ. 10.01.03-[ลำดับ]/[ปีการศึกษา]` |

### 7.6 ตัวอย่าง logic เงินยืมรองจ่าย

Template นี้ไม่มีคอลัมน์จำนวนเงินให้กรอกโดยตรงในชีท `ฎีกา` ที่ตรวจพบ แต่การใช้งานร่วมกับ Project Status ควรอธิบายหลักคิดดังนี้ และควรตรวจสอบสูตรในชีทจริงอีกครั้งหากมีการผูกยอดเงินจาก Project Status เพิ่มเติม

| กรณี | ยอดเบิกจริง | เคยยืมรองจ่าย | ผลลัพธ์ทางการเงิน | สรุปยอดเบิก |
|---|---:|---:|---:|---:|
| โครงการ A | 5,000 | 4,000 | เบิกเพิ่ม 1,000 | 5,000 |
| โครงการ B | 3,000 | 4,000 | คืนเงิน 1,000 | 3,000 |

อธิบายการใช้งาน

- ถ้ายอดเบิกจริงมากกว่ายอดที่เคยยืมรองจ่าย ให้จัดทำเอกสารเบิกเพิ่มส่วนต่าง
- ถ้ายอดเบิกจริงน้อยกว่ายอดที่เคยยืมรองจ่าย ให้จัดทำเอกสารคืนเงินส่วนต่าง
- ยอดสรุปเบิกควรเท่ากับยอดใช้จริงของโครงการ ไม่ใช่ยอดเงินที่เคยยืมรองจ่าย
- ก่อนออกเลขเอกสาร ควรตรวจยอดใน Project Status และเอกสารปิดโครงการให้ตรงกัน

### 7.7 ข้อควรระวัง

- ห้ามแก้ไขคอลัมน์ `A`, `B`, `E`, `F`, `G`, `H` หากเป็นข้อมูลที่สูตรดึงมา
- ห้ามแก้ไขคอลัมน์ซ่อน `I:S`
- หากเลขรันออกผิดหมวด ให้ตรวจข้อมูล `โครงการอยู่ภายใต้ฝ่าย` ใน Project Status
- หากสถานะขึ้น `ไม่พบข้อมูล` ให้ตรวจเลขรันและข้อมูลใน `SETTING`
- หากต้องแก้สถานะปิดโครงการ ควรแก้ที่ Project Status เป็นหลัก เพื่อให้ข้อมูลตรงกันทุกระบบ

## 8. ชีทระบบหลังบ้าน

### 8.1 ชีท `TEMPLATE`

`TEMPLATE` เป็นชีทต้นแบบสำหรับสร้างชีทเอกสารทั่วไปเพิ่มเติม โครงสร้างเหมือนกับชีทเอกสารทั่วไป

| คอลัมน์ | หัวข้อ | วิธีใช้งาน |
|---|---|---|
| `A` | ลำดับ | สูตรอัตโนมัติ ห้ามแก้ |
| `B` | เลขรัน | สูตรอัตโนมัติ ห้ามแก้ |
| `C` | รายการ | ผู้ใช้กรอกเอง |
| `D` | ผู้รับผิดชอบ | ผู้ใช้กรอกเอง |
| `E` | สถานะ | เลือกจากรายการสถานะ |
| `F` | หมายเหตุ | ผู้ใช้กรอกเอง |

หากต้องการเพิ่มชีทใหม่ ควร copy จาก `TEMPLATE` แล้วตั้งชื่อชีทใหม่ จากนั้นเลือกประเภทเอกสารใน `C1` ให้ตรงกับรายการในชีท `เลขรันเอกสาร`

ข้อควรระวัง

- ไม่ควรแก้ชีท `TEMPLATE` โดยตรงหากยังต้องใช้เป็นต้นแบบ
- หลัง copy ชีทใหม่ ต้องเพิ่มประเภทเอกสารและเลขฝ่ายในชีท `เลขรันเอกสาร` ด้วย
- ต้องตรวจสูตรเลขรันในคอลัมน์ `B` หลัง copy ว่ายังอ้างอิง `C1` และชีท `เลขรันเอกสาร` ถูกต้อง

### 8.2 ชีท `SETTING`

`SETTING` เป็นชีทระบบหลังบ้านที่ถูกซ่อนไว้ ใช้ดึงและจัดรูปข้อมูลจาก Project Status โดยพบสูตรสำคัญ เช่น

| ตำแหน่ง | สูตร/หน้าที่โดยสรุป |
|---|---|
| `A1` | ดึงข้อมูลจาก Project Status ช่วง `ตารางสถานะ!A2:F` |
| `H1` | ดึงข้อมูลจาก Project Status ช่วง `ตารางสถานะ!BI2:BK` |
| `M1` | ดึงข้อมูลจาก Project Status ช่วง `ตารางสถานะ!Y2:Z` |
| `O1` | กรองข้อมูลจากช่วง `M:N` |
| `L2` | กรองรหัสโครงการบางชุดเพื่อใช้ใน dropdown |
| `Q2` | แยกข้อมูลหลายบรรทัดด้วย `SPLIT` |
| `T2` | รวม/แปลงข้อมูลด้วย `QUERY`, `SPLIT`, `FLATTEN` |
| `X2` | รวมข้อมูลบางช่วงจากชีท `ฎีกา` |

ผู้ใช้ทั่วไปไม่ควรแก้ไขชีท `SETTING` เพราะอาจทำให้ระบบดึงข้อมูลผิด เลขรันผิด หรือ dropdown ในชีทเอกสารพิเศษไม่ทำงาน

ข้อควรระวัง

- ห้ามแก้สูตรใน `SETTING`
- ห้ามลบหรือย้ายคอลัมน์ใน `SETTING`
- หากขึ้น `#N/A` อาจเกิดจากยังไม่ได้อนุญาต `IMPORTRANGE`, ลิงก์ Project Status ไม่ถูกต้อง หรือ Project Status เปลี่ยนโครงสร้างคอลัมน์
- หาก Project Status เปลี่ยนชื่อชีทหรือเปลี่ยนคอลัมน์ ต้องตรวจสอบสูตรในชีทจริงอีกครั้ง

|---|---|---|
| เลขรันไม่ขึ้น | ยังไม่ได้กรอกช่องบังคับ | ตรวจช่องที่ต้องกรอก เช่น `รายการ`, `รหัสโครงการ`, `ผู้รับผิดชอบ`, `สถานะ` |
| ขึ้น `กรอกข้อมูลไม่ครบถ้วน` | ชีท `เอกสารยืมรองจ่าย` กรอกข้อมูลไม่ครบ `D`, `H`, `I` | กรอกให้ครบทั้ง 3 ช่อง |
| ขึ้น `ไม่พบข้อมูล` | รหัสโครงการหรือเลขรันไม่ตรงกับข้อมูลใน `SETTING` | ตรวจ Project Status และตรวจว่าข้อมูลถูก import แล้ว |
| ชื่อโครงการไม่ขึ้น | ลิงก์ Project Status ไม่ถูกต้อง หรือยังไม่ได้อนุญาต `IMPORTRANGE` | ตรวจ `เลขรันเอกสาร!B2` และกดอนุญาตการเชื่อมข้อมูลใน Google Sheets |
| เลขรันปีผิด | ปีการศึกษาใน `B1` ไม่ถูกต้อง | แก้ปีการศึกษาในชีท `เลขรันเอกสาร!B1` |
| เลขรันซ้ำ | มีการลบแถวเก่า หรือแก้สูตร/เลขรันเอง | ไม่ควรลบแถวเอกสาร ให้ใช้สถานะ `ยกเลิก` และเก็บเลขเดิมไว้ |
| Dropdown รหัสโครงการว่าง | `SETTING` ดึงข้อมูลไม่ได้ | ตรวจลิงก์ Project Status และสูตรใน `SETTING` |
| สูตรกลายเป็นข้อความหรือหาย | มีการ copy/paste ทับสูตร | กู้สูตรจากไฟล์ Template หรือ copy แถวสูตรจากแถวที่ถูกต้อง |
| ชีทใหม่ออกเลขผิดหมวด | ยังไม่ได้เลือกประเภทเอกสารใน `C1` หรือยังไม่ได้เพิ่มรายการในชีท `เลขรันเอกสาร` | ตรวจ `C1` และตารางประเภทเอกสารในชีทหน้าหลัก |

## 10. ข้อควรระวังสำคัญ

1. ห้ามแก้ไขเลขรันด้วยมือ เว้นแต่ได้รับอนุมัติและมีบันทึกเหตุผล
2. ห้ามลบแถวที่เคยออกเลขแล้ว ให้เปลี่ยนสถานะเป็น `ยกเลิก` แทน
3. ห้ามเปลี่ยนหัวคอลัมน์ เพราะสูตรและ dropdown อ้างอิงตำแหน่งคอลัมน์เดิม
4. ห้ามลบสูตรในแถวที่ 4 ซึ่งเป็นจุดเริ่มต้นของสูตรแบบ array หลายชีท
5. ห้ามแก้ชีท `SETTING` หากไม่เข้าใจสูตรหลังบ้าน
6. ก่อนใช้งานจริงทุกปี ต้องตั้งค่า `ปีการศึกษา` และ `Project Status` ให้ถูกต้อง
7. เมื่อ Project Status เปลี่ยนโครงสร้าง ควรตรวจสอบสูตรในชีทจริงอีกครั้ง
8. ควรจำกัดสิทธิ์แก้ไขไฟล์ให้เฉพาะผู้รับผิดชอบ เพื่อป้องกันสูตรเสียหาย
