# คู่มือการใช้งานระบบ Project Status ฝ่ายเหรัญญิก SGCU

> คู่มือนี้จัดทำจากไฟล์ `SUCU68 Project Status.xlsx` โดยอ่านทุกชีทในไฟล์ ได้แก่ `ตารางสถานะ`, `CONTACT`, `ตรวจสอบงานผู้ช่วยเหรัญญิก` และ `ประธาน 4 ฝ่าย ติดตามการส่งเอกสา` ข้อมูลสูตรบางส่วนในไฟล์ Excel แสดงเป็น `__xludf.DUMMYFUNCTION(...)` เนื่องจากเป็นสูตรจาก Google Sheets ที่ Excel ไม่รองรับโดยตรง เช่น `IMPORTRANGE`, `QUERY`, `FILTER`, `MAP` และ `VSTACK` ดังนั้นกรณีต้องแก้สูตรจริง ควรตรวจสอบในชีท Google Sheets ต้นทางเพิ่มเติม

## 1. ภาพรวมของระบบ

Project Status เป็นฐานข้อมูลกลางสำหรับติดตามสถานะโครงการและงานเอกสารของฝ่ายเหรัญญิก SGCU โดยชีทหลักคือ `ตารางสถานะ` ซึ่งเก็บข้อมูลโครงการตั้งแต่ข้อมูลพื้นฐาน การอนุมัติโครงการ การโอนงบประมาณ การยืมรองจ่าย การส่งเอกสารปิดโครงการ งบประมาณที่ใช้จริง งบคงเหลือ และเลขเอกสารที่เชื่อมกับระบบอื่น

ระบบนี้ใช้เพื่อ

- ติดตามสถานะการอนุมัติโครงการและงบประมาณ
- ติดตามการโอนงบประมาณและส่วนต่างจากการเปลี่ยนเพดานงบ
- ติดตามการยืมรองจ่ายและยอดที่ต้องเบิกเพิ่มหรือคืนเงิน
- ติดตามกำหนดส่งเอกสารสรุป/ปิดโครงการ
- สร้างข้อความติดตามเอกสารจากสถานะและวันครบกำหนด
- เชื่อมข้อมูลเลขรันเอกสารจากระบบเอกสารอื่น เช่น เอกสารเปลี่ยนเพดานงบ เอกสารยืมรองจ่าย และฎีกา
- ทำหน้าที่เป็น source สำหรับชีทตรวจงานผู้ช่วยเหรัญญิกและชีทติดตามของประธาน 4 ฝ่าย

## 2. วิธีเริ่มต้นใช้งาน

### 2.1 เปิดไฟล์และใช้งาน filter

ชีทที่ควรใช้เป็นหลักคือ `ตารางสถานะ` โดยไฟล์มี filter ตั้งแต่แถวหัวตาราง:

- `ตารางสถานะ`: ช่วง filter `$A$2:$BZ$551`
- `ตรวจสอบงานผู้ช่วยเหรัญญิก`: ช่วง filter `$A$2:$CA$546`
- `ประธาน 4 ฝ่าย ติดตามการส่งเอกสา`: ช่วง filter `$A$2:$G$544`

เมื่อต้องการค้นหาโครงการ ให้ใช้ filter ที่หัวคอลัมน์ เช่น รหัสโครงการ ชื่อโครงการ ผู้รับผิดชอบ ฝ่าย สถานะปิดโครงการ หรือผู้สอบตรวจสอบเอกสาร

### 2.2 การเลือกปีการศึกษา

จากข้อมูลใน workbook ไม่พบช่องเลือกปีการศึกษาแบบ dropdown หรือ selector โดยตรง ข้อมูลปีปรากฏอยู่ในรหัสเอกสารและชื่อโครงการ เช่น `2568`, `68`, และเลขรันเอกสาร หากต้องการรองรับหลายปีการศึกษา ควรตรวจสอบในชีทจริงเพิ่มเติมว่ามีการแยกไฟล์หรือแยก Google Sheet ตามปีหรือไม่

### 2.3 การกรอกข้อมูลใหม่

ให้เพิ่มข้อมูลใน `ตารางสถานะ` เป็นหลัก โดยกรอกเฉพาะคอลัมน์ที่เป็น input ส่วนคอลัมน์ formula/auto ไม่ควรแก้ไขเอง ยกเว้นมีผู้ดูแลระบบยืนยันให้แก้สูตร

แนวทางเพิ่มแถว:

- เพิ่มข้อมูลแบบ append ภายในช่วงที่สูตรและ validation ครอบคลุมอยู่ เช่นไม่ควรเพิ่มข้อมูลนอกช่วงที่สูตรอ้างถึง `3:551`
- หากต้องเพิ่มแถวเกินช่วงเดิม ต้องขยายสูตร array, data validation และ filter ให้ครอบคลุมแถวใหม่ด้วย
- ห้าม sort เฉพาะบางคอลัมน์ ให้ sort ทั้งตารางพร้อมกันเสมอ
- ห้ามแก้ชื่อหัวคอลัมน์ เพราะสูตร `XLOOKUP`, `FILTER`, `IMPORTRANGE` และชีทปลายทางพึ่งพาโครงสร้างตารางเดิม

## 3. โครงสร้างชีทและความสัมพันธ์

### 3.1 ชีท `ตารางสถานะ`

ชีทนี้เป็นฐานข้อมูลหลักของระบบ ใช้โดยฝ่ายเหรัญญิก ผู้ช่วยเหรัญญิก และผู้ดูแลข้อมูลโครงการ ข้อมูลจากชีทนี้ถูกส่งต่อไปยัง `ตรวจสอบงานผู้ช่วยเหรัญญิก` และ `ประธาน 4 ฝ่าย ติดตามการส่งเอกสา` ผ่านสูตร `FILTER` และ `XLOOKUP`

ข้อมูลในชีทนี้แบ่งเป็นกลุ่มใหญ่:

- ข้อมูลโครงการ
- อนุมัติโครงการและงบประมาณ
- โอนงบประมาณ
- ยืมรองจ่าย
- ส่งปิดโครงการ
- สรุปงบประมาณ
- DATA โอนงบประมาณ
- DATA ยืมรองจ่าย
- DATA ฎีกา

### 3.3 ชีท `CONTACT`

### 3.4 ชีท `ตรวจสอบงานผู้ช่วยเหรัญญิก`

ชีทนี้เป็นหน้าตรวจงานรายผู้ช่วยเหรัญญิก โดยเซลล์ `A1` ระบุชื่อผู้ช่วย เช่นในไฟล์พบค่า `แม็ค` และสูตรหลักกรองข้อมูลจาก `ตารางสถานะ` เฉพาะรายการที่มีชื่อในคอลัมน์ผู้รับผิดชอบ/เหรัญญิกโครงการ/ผู้สอบตรวจสอบเอกสาร และมีเงื่อนไขเกี่ยวกับการติดตามเอกสาร

ลักษณะข้อมูล:

- เป็นชีทปลายทางที่ดึงข้อมูลจาก `ตารางสถานะ`
- คอลัมน์ส่วนใหญ่เป็น formula ไม่ใช่ source of truth
- โครงสร้างคอลัมน์แทบเหมือน `ตารางสถานะ` ตั้งแต่ A ถึง BK
- มี validation ติดมาหลายช่วง แต่จากสูตรที่พบ การแก้ไขข้อมูลควรทำที่ `ตารางสถานะ` เป็นหลัก

Workflow ของชีทนี้:

1. ผู้ดูแลระบุชื่อผู้ช่วยที่ `A1`
2. สูตร `FILTER` ดึงโครงการที่เกี่ยวข้องกับชื่อนั้นจาก `ตารางสถานะ`
3. ผู้ช่วยใช้ดูรายการที่ต้องติดตาม ตรวจเอกสาร หรือดำเนินการต่อ
4. หากต้องแก้สถานะหรือยอดเงิน ควรกลับไปแก้ที่ `ตารางสถานะ`

### 3.5 ชีท `ประธาน 4 ฝ่าย ติดตามการส่งเอกสา`

ชีทนี้เป็นหน้าสรุปรายการติดตามเอกสารสำหรับประธาน 4 ฝ่าย โดยดึงจาก `ตารางสถานะ` เฉพาะโครงการที่ไม่ใช่ `องค์การบริหารสโมสรนิสิต`, ไม่ใช่ `องค์การบริหารสโมสรนิสิต, สภานิสิต`, และไม่ใช่ `สภานิสิต` รวมทั้งมีสถานะปิดโครงการเป็น `รอส่งเอกสารปิดโครง`, `ส่งให้ชมรมกลับเพื่อแก้ไข`, หรือ `ชมรมรับเอกสารแก้ไข` และมีเงื่อนไขวันที่จากสูตร `TODAY()+1 > วันที่ปฏิบัติงานวันสุดท้าย`

## 5. ระบบงบประมาณ

ระบบงบประมาณในไฟล์ใช้คอลัมน์หลักดังนี้:

- `O` งบประมาณที่ได้รับอนุมัติ (100%) คือวงเงินตั้งต้น
- `U` ระบุว่าส่วนต่างจากการโอนงบเป็น `เพิ่ม` หรือ `ลด`
- `V` จำนวนเงินส่วนต่าง
- `W` งบประมาณสุทธิหลังปรับเพดาน
- `AJ` จำนวนเงินใช้จริง
- `BE` งบประมาณที่ใช้จริง ซึ่งเท่ากับ AJ หากมีค่า มิฉะนั้นเป็น 0
- `BF` งบประมาณคงเหลือ
- `BG` ร้อยละการใช้งบประมาณ

Logic การคำนวณ:

- หากมีการลดงบ: `งบประมาณสุทธิ = งบอนุมัติ - ส่วนต่าง`
- หากมีการเพิ่มงบ: `งบประมาณสุทธิ = งบอนุมัติ + ส่วนต่าง`
- หากสถานะโอนงบประมาณอยู่ในกลุ่มที่กระบวนการมีผล ระบบใช้งบประมาณสุทธิเป็นฐานคำนวณงบคงเหลือ
- หากสถานะโอนงบประมาณยังไม่มีผล ระบบใช้งบประมาณที่ได้รับอนุมัติเป็นฐานคำนวณงบคงเหลือ
- `%การใช้งบประมาณ` คำนวณจากยอดใช้จริงหารด้วยฐานงบที่เกี่ยวข้อง

## 6. ระบบยืมรองจ่าย

ระบบยืมรองจ่ายใช้ข้อมูลหลักจากคอลัมน์ Y ถึง AE และ BB ถึง BC

- `Y` สถานะยืมรองจ่าย
- `Z` เลขรันเอกสารยืมรองจ่าย
- `AC` วันที่ต้องคืนรองจ่าย
- `AD` ร้อยละการยืมรองจ่าย
- `AE` จำนวนเงินยืมรองจ่าย
- `AJ` จำนวนเงินใช้จริง
- `BB` ระบุว่าต้อง `เบิก` หรือ `คืน`
- `BC` จำนวนเงินส่วนต่าง

Logic ที่พบในไฟล์:

- `AD` ดึงสัดส่วนจาก `BR`
- `AE = O * AD / 100`
- หากสถานะยืมรองจ่ายเป็น `โครงการรับเงินแล้ว`, `เหรัญญิกรับเงินจากกิจการนิสิต` หรือ `ส่งกิจการนิสิตแล้ว รออนุมัติ` ระบบคำนวณ `BC = AJ - AE`
- หาก `BC > 0` ระบบแสดง `เบิก`
- หาก `BC < 0` ระบบแสดง `คืน`
- หาก `BC = 0` ไม่แสดงข้อความเบิก/คืน

ตัวอย่างการใช้งาน:

- โครงการ A ยืมรองจ่าย 4,000 บาท และใช้จริง 5,000 บาท: `BC = 5,000 - 4,000 = 1,000` ระบบแสดง `เบิก` หมายถึงต้องเบิกเพิ่ม 1,000 บาท
- โครงการ B ยืมรองจ่าย 4,000 บาท และใช้จริง 3,000 บาท: `BC = 3,000 - 4,000 = -1,000` ระบบแสดง `คืน` หมายถึงต้องคืนเงิน 1,000 บาท

คอลัมน์ `BY:BZ` ดึงข้อมูลจากชีท `ฎีกา` และคอลัมน์ `BI` ใช้ `XLOOKUP` หาเลขฎีกาตามรหัสโครงการเมื่อมีจำนวนเงินใช้จริง

### 7.5 Dashboard และหน้าเว็บไซต์

ไม่พบชีท dashboard โดยตรงใน workbook นี้ แต่ `ตารางสถานะ` มีคอลัมน์สรุปที่พร้อมนำไปใช้ต่อ ได้แก่ งบใช้จริง งบคงเหลือ %การใช้งบประมาณ ระยะเวลาส่งสรุปโครงการ สถานะปิดโครงการ และเลขฎีกา หากเว็บไซต์หรือ dashboard ต้องการแสดงผล ควรดึงจาก `ตารางสถานะ` เป็นหลัก ไม่ควรดึงจากชีทตรวจงานซึ่งเป็นชีทกรองข้อมูลเฉพาะบริบท

## 8. ข้อควรระวัง

- ห้ามลบหรือแก้สูตรในคอลัมน์ auto/formula เช่น A, R, W, AC:AE, AK, AP, AV, AW, BB:BC, BE:BI, BJ:BK และ data columns BM:BZ
- ห้ามแก้ชื่อหัวคอลัมน์หรือย้ายคอลัมน์ เพราะชีทอื่นอ้างอิงตำแหน่งคอลัมน์โดยตรง
- ห้าม sort เฉพาะบางคอลัมน์ ให้เลือกทั้งตารางก่อน sort
- ควรกรอกข้อมูลใหม่แบบ append และตรวจว่าช่วงสูตร/validation ครอบคลุมแถวใหม่
- ควรตรวจสอบสิทธิ์ `IMPORTRANGE` หากข้อมูลเลขรันไม่ขึ้น
- ช่องวันที่มี custom validation ควรกรอกเป็นวันที่จริง ไม่ใช่ข้อความ
- ชีท `ตรวจสอบงานผู้ช่วยเหรัญญิก` และ `ประธาน 4 ฝ่าย ติดตามการส่งเอกสา` เป็นชีทแสดงผลจากสูตร ควรแก้ข้อมูลต้นทางที่ `ตารางสถานะ`
- หากใช้งานใน Excel สูตรจาก Google Sheets บางสูตรอาจแสดงเป็น `__xludf.DUMMYFUNCTION`; ควรใช้งานหรือแก้ไขสูตรบน Google Sheets ต้นทาง

## 9. ปัญหาที่พบบ่อยและวิธีแก้

| ปัญหา | สาเหตุที่เป็นไปได้ | วิธีแก้ |
|---|---|---|
| เลขรันเอกสารโอนงบ/ยืมรองจ่ายไม่ขึ้น | `IMPORTRANGE` ยังไม่ได้รับสิทธิ์ หรือรหัสโครงการไม่ตรงกัน | เปิด Google Sheets ต้นทาง กดอนุญาตการเชื่อมต่อ และตรวจรหัสโครงการ |
| สูตรแสดง `__xludf.DUMMYFUNCTION` | เปิดไฟล์ใน Excel ซึ่งไม่รองรับสูตร Google Sheets บางสูตร | ใช้งานบน Google Sheets หรือแก้สูตรในชีทจริง |
| dropdown หาย | เพิ่มแถวนอกช่วง validation หรือ copy/paste แบบไม่รักษา validation | ขยาย validation ให้ครอบคลุมแถวใหม่ |
| status ไม่ update ในชีทตรวจงาน | แก้ข้อมูลในชีทปลายทางแทนชีทต้นทาง หรือสูตรกรองไม่เข้าเงื่อนไข | แก้ที่ `ตารางสถานะ` และตรวจเงื่อนไขสูตร FILTER |
| งบคงเหลือผิด | สถานะโอนงบประมาณยังไม่อยู่ในกลุ่มที่สูตรถือว่ามีผล หรือกรอก U/V ไม่ครบ | ตรวจ Q, U, V, O, W และ AJ |
| ระบบไม่แสดงเบิก/คืนรองจ่าย | ยังไม่มีสถานะปิดโครงฎีกา หรือ BC เป็น 0/ว่าง | ตรวจ Y, AE, AJ, BA และ BC |
| ข้อความติดตามไม่ขึ้น | ยังไม่เข้าเงื่อนไขระยะเวลา หรือสถานะปิดโครงการไม่ตรง | ตรวจ AW, AZ, AL:AO, AQ และ AR:AU |

## 10. Workflow แนะนำสำหรับผู้ใช้งาน

### 10.1 เปิดโครงการใหม่

1. เพิ่มแถวใน `ตารางสถานะ`
2. กรอก `รหัสโครงการ`, `ชื่อโครงการ`, `ผู้รับผิดชอบโครงการ`, `ฝ่ายที่รับผิดชอบโครงการ`
3. กรอกสถานะอนุมัติในคอลัมน์ H
4. กรอกข้อมูลที่ประชุม วันที่อนุมัติ วันที่ปฏิบัติงานวันสุดท้าย แหล่งงบ และงบอนุมัติ
5. ตรวจว่าเลขลำดับในคอลัมน์ A และสูตรที่เกี่ยวข้องทำงาน

### 10.2 ติดตามการโอนงบประมาณ

1. เลือกสถานะในคอลัมน์ Q
2. หากมีการเพิ่ม/ลดงบ ให้กรอก U และ V
3. ตรวจคอลัมน์ W ว่างบประมาณสุทธิคำนวณถูกต้อง
4. ตรวจคอลัมน์ R ว่าเลขรันเอกสารโอนงบประมาณถูกดึงมาหรือไม่

### 10.3 ติดตามการยืมรองจ่าย

1. เลือกสถานะในคอลัมน์ Y
2. ตรวจเลขรันในคอลัมน์ Z
3. ตรวจร้อยละการยืมใน AD และจำนวนเงินยืมใน AE
4. หากต้อง override สัดส่วน ให้ใช้ช่อง BS โดยควรให้ผู้ดูแลระบบตรวจสอบก่อน

### 10.4 ปิดโครงการ

1. กรอกจำนวนเงินใช้จริงใน AJ
2. ตรวจวันครบกำหนดส่งเอกสารใน AK และวันคงเหลือใน AW
3. อัปเดตสถานะปิดโครงการใน AZ ตามขั้นตอนจริง
4. หากส่งกลับแก้ไข ให้กรอก AQ และใช้สถานะ `ส่งให้ชมรมกลับเพื่อแก้ไข` หรือ `ชมรมรับเอกสารแก้ไข`
5. เมื่อส่งกิจการนิสิตเรียบร้อย ให้เปลี่ยน AZ เป็น `ส่งกิจการนิสิตเรียบร้อย`
6. อัปเดต BA ตามสถานะฎีกา/การเงิน
7. ตรวจ BB และ BC เพื่อดูว่าต้องเบิกเพิ่มหรือคืนรองจ่าย

### 10.5 ตรวจสอบสรุปงบประมาณ

1. ตรวจ BE ว่าตรงกับยอดใช้จริง
2. ตรวจ BF ว่างบคงเหลือคำนวณจากฐานงบที่ถูกต้อง
3. ตรวจ BG ว่าสัดส่วนการใช้งบสมเหตุสมผล
4. ตรวจ BI ว่าเลขฎีกาดึงขึ้นเมื่อมีจำนวนเงินใช้จริง

## 11. สรุปสำหรับผู้ดูแลระบบ

ระบบ Project Status ในไฟล์นี้ออกแบบให้ `ตารางสถานะ` เป็นฐานข้อมูลกลาง ส่วนชีทอื่นเป็นมุมมองสำหรับตรวจงานและติดตามเอกสาร ผู้ดูแลระบบควรรักษาโครงสร้างคอลัมน์ สูตร array และช่วง validation ให้คงเดิม โดยเฉพาะช่วงข้อมูลที่อ้างอิงถึงแถว 551/546/544 หากมีการขยายจำนวนโครงการ ต้องขยายสูตรและ validation พร้อมกัน มิฉะนั้นข้อมูลใหม่อาจไม่ถูกดึงไปยังระบบเลขรันเอกสาร ชีทตรวจงาน หรือหน้าติดตามของประธาน 4 ฝ่าย
