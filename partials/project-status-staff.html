Opening note

ข้อความเปิดเล่ม

ขอแสดงความยินดีกับผู้ที่ได้รับหน้าที่เหรัญญิกและผู้ร่วมงานทุกคน ขอให้การทำงานตลอดวาระเต็มไปด้วยความสุข ความภาคภูมิใจ และความตั้งใจที่จะดูแลงบประมาณและทรัพย์สินของนิสิตอย่างดีที่สุด

ขอให้ทุกคนสามารถดำเนินงานได้อย่างราบรื่น จบวาระด้วยการปิดโครงการครบถ้วน โปร่งใส และส่งต่องานให้รุ่นถัดไปได้อย่างสมบูรณ์ คู่มือฉบับนี้จึงตั้งใจเขียนให้เป็นทั้งภาพรวม วิธีคิด และแนวทางเริ่มต้นงานที่นำไปใช้ได้จริงในช่วงแรกของการรับตำแหน่ง

First steps

ขั้นตอนหลังการเลือกตั้ง

  1. 1

    ประชุม 10 ตำแหน่งส่วนกลาง

    หลังประกาศผลเลือกตั้ง จะมีการประชุม 10 ตำแหน่งส่วนกลางเพื่อเริ่มวางระบบการทำงานร่วมกัน

  2. 2

    วางนโยบายและแบ่งฝ่ายย่อย

    วาระสำคัญคือการวางนโยบาย ทิศทางการทำงาน และแบ่งฝ่ายย่อยที่แต่ละตำแหน่งจะรับผิดชอบ

  3. 3

    ยกฝ่ายเหรัญญิกขึ้นพิจารณาเร็ว

    โดยปกติฝ่ายเหรัญญิกมักถูกยกขึ้นพิจารณาเป็นฝ่ายแรก เพราะมีภาระงานสูงและต้องใช้ความรู้เฉพาะ

  4. 4

    เสนอแต่งตั้งประธานฝ่ายย่อย

    เหรัญญิกควรใช้ช่วงประชุมแรกในการเสนอให้แต่งตั้งประธานฝ่ายย่อยของตนโดยเร็ว

การแต่งตั้งประธานฝ่ายย่อย

เหรัญญิกควรต่อรองและเสนอเหตุผลให้แต่งตั้งฝ่ายย่อยของตนก่อน โดยชี้ให้เห็นว่างานเหล่านี้มีผลต่อการเปิดระบบงบประมาณและโครงการใหญ่ช่วงต้นปีโดยตรง

Interview guide

คำถามสัมภาษณ์

ใช้เป็นชุดคำถามตั้งต้น สามารถปรับน้ำหนักตามลักษณะงานและจำนวนผู้สมัครในแต่ละรอบได้

เหรัญญิก
  1. เคยทำเอกสารการเงินหรือเอกสารเบิกจ่ายมาก่อนหรือไม่
  2. หากเอกสารโครงการส่งมาผิดหลายจุด จะจัดการอย่างไร
  3. ถ้าฝ่ายกิจกรรมต้องการเงินด่วน แต่เอกสารยังไม่ครบ จะทำอย่างไร
  4. คุณรับมือกับงานละเอียดและซ้ำ ๆ ได้ดีแค่ไหน
  5. ถ้ามีโครงการใหญ่ 3 โครงการเข้ามาพร้อมกัน จะจัดลำดับงานอย่างไร
  6. เคยใช้ Google Sheets หรือ Excel ระดับใด
  7. หากต้องทวงเอกสารหรือเงินคืนจากเพื่อน จะสื่อสารอย่างไร
  8. คุณคิดว่าความโปร่งใสทางการเงินสำคัญอย่างไร
  9. หากพบตัวเลขผิดหลังอนุมัติไปแล้ว จะทำอย่างไร
  10. ทำไมถึงอยากทำงานฝ่ายเหรัญญิก
บริหารและพัฒนางบประมาณ
  1. คุณคิดว่าการตัดงบควรพิจารณาจากอะไร
  2. หากโครงการเสนอของบสูงเกินความจำเป็น จะอธิบายอย่างไร
  3. ถ้าฝ่ายที่ถูกตัดงบไม่พอใจ คุณจะรับมืออย่างไร
  4. คุณแยก “จำเป็น” กับ “อยากได้” ในงบประมาณอย่างไร
  5. เคยวิเคราะห์ข้อมูลหรือตารางงบประมาณมาก่อนหรือไม่
  6. ถ้างบรวมมีจำกัด แต่ทุกฝ่ายบอกว่าจำเป็น จะจัดลำดับอย่างไร
  7. คุณคิดว่าความยุติธรรมในการจัดสรรงบคืออะไร
  8. ถ้าต้องอ่านงบจำนวนมากในเวลาจำกัด จะจัดการอย่างไร
  9. มีแนวคิดอะไรในการพัฒนาระบบติดตามงบประมาณ
  10. ทำไมถึงสนใจงานบริหารและพัฒนางบประมาณ
หาทุนและสิทธิประโยชน์
  1. เคยติดต่อผู้สนับสนุนหรือองค์กรภายนอกมาก่อนหรือไม่
  2. หากผู้สนับสนุนปฏิเสธ จะทำอย่างไรต่อ
  3. คุณคิดว่า sponsor ต้องการอะไรจากกิจกรรมนิสิต
  4. ถ้าต้องเขียน proposal ขอสนับสนุน จะเน้นประเด็นใด
  5. คุณสื่อสารกับคนแปลกหน้าได้ดีแค่ไหน
  6. หากผู้สนับสนุนเสนอเงื่อนไขที่ไม่เหมาะสม จะจัดการอย่างไร
  7. คุณคิดว่าสิทธิประโยชน์ที่ดีควรเป็นอย่างไร
  8. ถ้ามีเวลาจำกัดก่อนโครงการใหญ่ จะหาทุนอย่างไร
  9. เคยทำงานขาย ประสานงาน หรือเจรจามาก่อนหรือไม่
  10. ทำไมถึงอยากทำงานหาทุนและสิทธิประโยชน์
กายภาพและพัสดุ
  1. คุณเคยดูแลของส่วนกลางหรือคลังพัสดุมาก่อนหรือไม่
  2. หากพัสดุเสียหายหลังคืน จะจัดการอย่างไร
  3. ถ้ามีคนขอยืมของด่วนโดยไม่ทำตามขั้นตอน จะทำอย่างไร
  4. คุณคิดว่าการตรวจนับพัสดุควรทำอย่างไรให้แม่นยำ
  5. เคยเปรียบเทียบราคาหรือหาราคากลางมาก่อนหรือไม่
  6. ถ้าราคากลางของเดิมไม่ตรงกับราคาปัจจุบัน จะทำอย่างไร
  7. หากคลังพัสดุไม่เป็นระบบ คุณจะเริ่มแก้จากอะไร
  8. คุณรับงานที่ต้องใช้แรง ตรวจของ หรือจัดของได้หรือไม่
  9. ถ้าฝ่ายอื่นทำของหายแต่ไม่ยอมรับผิด จะทำอย่างไร

Core mission

ภารกิจหลักของฝ่ายบริหารและพัฒนางบประมาณ

การพิจารณางบประมาณถือเป็นภารกิจหลักของฝ่ายบริหารและพัฒนางบประมาณ โดยจะมีผู้ช่วยเหรัญญิกเข้ามาช่วยดำเนินงานในหลายส่วน เนื่องจากเป็นกระบวนการที่ใช้เวลานาน ต้องอาศัยความละเอียด และเกี่ยวข้องกับทุกองค์กรภายในสโมสรนิสิต

  • ควบคุมงบประมาณให้อยู่ภายในเพดาน
  • กระจายทรัพยากรอย่างเหมาะสม
  • ทำให้ทุกองค์กรสามารถดำเนินโครงการได้
  • ป้องกันการใช้งบประมาณเกินความจำเป็น
  • สร้างความโปร่งใสในการใช้งบประมาณนิสิต

3. งานที่ต้องทำเป็นรอบ

แบ่งงานเป็นรอบสั้น ๆ จะไม่หลุดคิวเอกสารและรู้ทันว่าโครงการไหนต้องตามต่อ

  1. ทุกวันดูโครงการที่สถานะเปลี่ยน, งานรอตรวจ, งานใกล้ครบกำหนด และข้อความจากฝ่าย/ชมรม
  2. ทุกสัปดาห์ตรวจยอดงบใช้จริง, งานยืมรองจ่าย, งานรอส่งกิจการนิสิต และงานปิดโครงการค้าง
  3. ทุกเดือนexport รายการปิดโครงการ, ตรวจเลขฎีกา, เทียบเลขรันกับ Project Status และสำรองไฟล์

ปิดโครงการและ Export CSV

ใช้เมื่อต้องสรุปโครงการที่ถึงรอบปิด ทำเอกสาร Word หลายโครงการพร้อมกัน และตรวจยอดเบิก/คืนก่อนส่งต่อ

1 แถว = 1 โครงการ แปลง CSV เป็น XLSX ก่อน merge
Source Project Status ต้องกรอกงบใช้จริง วันที่ส่งปิด สถานะปิด และเลขฎีกาให้เรียบร้อยก่อน export
Output CSV/XLSX + Word Template CSV จากเว็บใช้เป็นฐานข้อมูล ส่วน Word Template ใช้สร้างเอกสารปิดโครงการแบบ Mail Merge

ทำตามลำดับนี้เมื่อถึงรอบปิดโครงการ

  1. 1
    กรองโครงการที่ต้องปิด ใน Project Status ดูโครงการที่วันปฏิบัติงานสิ้นสุดแล้วหรือครบกำหนดส่งเอกสาร แยกงานเป็น รอเอกสารจากฝ่าย/ชมรม, รอตรวจ, รอส่งกิจการนิสิต, ส่งเรียบร้อย
  2. 2
    กรอกข้อมูลปิดให้ครบ กรอกงบใช้จริง วันที่ส่งปิด และสถานะปิดโครงการใน Project Status ตรวจยอดเบิกเพิ่ม/คืนเงินจากช่องคำนวณก่อนออกเอกสาร
  3. 3
    ตรวจเลขฎีกา ออกเลขใน Template เลขรันเอกสาร แล้วดูว่าเลขกลับมาแสดงใน Project Status ถ้าเลขไม่ขึ้น ให้ตรวจรหัสโครงการและสิทธิ์ IMPORTRANGE
  4. 4
    Export CSV จากเว็บ เลือกเดือนสุดท้ายที่ต้องการ export หรือปล่อยเป็นทั้งหมด กด Export CSV ปิดโครงการ แล้วเปิดไฟล์เพื่อตรวจหัวคอลัมน์และจำนวนแถว
  5. 5
    แปลง CSV เป็น XLSX ใน Excel: เปิด CSV แล้วกด File > Save As > Excel Workbook (*.xlsx) ใน Google Sheets: อัปโหลด CSV แล้วกด File > Download > Microsoft Excel (.xlsx)
  6. 6
    ทำ Mail Merge ใน Word เปิด Word Template แล้วเลือก Mailings > Start Mail Merge > Letters กด Select Recipients > Use an Existing List แล้วเลือกไฟล์ XLSX Preview ทีละรายการ ก่อน Finish & Merge และ export เป็น PDF
คำสั่งหลักใน Word Mailings > Start Mail Merge > Letters Mailings > Select Recipients > Use an Existing List Mailings > Insert Merge Field Mailings > Preview Results Mailings > Finish & Merge
เช็กก่อนส่งไฟล์ปิดโครงการ ชื่อโครงการ หน่วยงาน เลขฎีกา ยอดเงิน และบัญชีต้องตรงกับ Project Status แถวแรกของ XLSX ต้องเป็นชื่อคอลัมน์ ห้าม merge cell และควรเก็บข้อมูลไว้ใน sheet เดียว หลังเชื่อมกับ Word แล้ว อย่าย้ายไฟล์ XLSX เพราะ Word จะจำ path ของไฟล์ไว้ Microsoft Support: Mail Merge
Actions เครื่องมือที่ใช้ในหน้านี้ ทำข้อมูลใน Project Status ให้ครบก่อน แล้วค่อยดาวน์โหลด template และ export CSV
1. ดาวน์โหลด Word Template ดาวน์โหลด Template ได้ไฟล์ .docx สำหรับใช้เป็น Word Template ของเอกสารปิดโครงการ
2. Export CSV จากระบบ
ค่าเริ่มต้น export ทุกเดือน โดยอ้างอิงกำหนดปิดโครงการ
ได้ไฟล์ .csv ที่มีข้อมูลโครงการ ผู้ตรวจ สถานะปิด งบใช้จริง และข้อมูลบัญชีสำหรับ Mail Merge
3. หลัง export ต้องตรวจ จำนวนแถวใน CSV เท่ากับจำนวนโครงการที่จะออกเอกสาร ไม่มีช่องเลขฎีกา ยอดเงิน หรือบัญชีว่างในรายการที่ต้องส่งจริง ชื่อคอลัมน์ยังอ่านได้ใน Excel/Word ไม่กลายเป็นอักขระเพี้ยน

เลขรันเอกสาร

ระบบเลขรันคือสมุดทะเบียนเอกสารของฝ่ายเหรัญญิก ใช้คุมว่าเอกสารใดออกเลขแล้ว อยู่สถานะไหน และเชื่อมกลับไป Project Status อย่างไร

1 ทำสำเนาไฟล์สำหรับปีใหม่

เปิด Google Sheets Template เลขรันเอกสาร แล้วกด File > Make a copy จากนั้นตั้งชื่อไฟล์ตามปีการศึกษา เช่น SGCU68-10 เลขรันเอกสาร

เข้าไปที่ชีท เลขรันเอกสาร กรอกปีการศึกษาใน B1 และวางลิงก์ Google Sheets Project Status ของปีนั้นใน B2

ลองเลือกโครงการในชีทเอกสารยืมรองจ่ายหรือฎีกา ถ้าชื่อโครงการ หน่วยงาน หรือฝ่ายไม่ขึ้น ให้ตรวจสิทธิ์ IMPORTRANGE ก่อนออกเลขจริง
