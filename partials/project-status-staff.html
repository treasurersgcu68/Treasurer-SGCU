Export CSV ปิดโครงการ

ใช้ไฟล์จากระบบไปสร้างเอกสารปิดโครงการใน Microsoft Word แบบอัตโนมัติ

1 row = 1 document ใช้ XLSX เป็นหลัก
Data source ฐานข้อมูล ใช้ไฟล์ CSV ที่ export จากระบบ แล้วแปลงเป็น mailmerge.xlsx
Document Word Template ใช้ไฟล์ template.docx ที่ดาวน์โหลดจากปุ่มด้านขวา
  1. 1
    เตรียมไฟล์ 2 ไฟล์ กด ดาวน์โหลด Template เพื่อได้ไฟล์ Word เลือก Export ถึงเดือน ถ้าต้องการจำกัดช่วงเวลา แล้วกด Export CSV ปิดโครงการ
  2. 2
    แปลง CSV เป็น XLSX ใน Excel: เปิด CSV แล้วกด File > Save As > Excel Workbook (*.xlsx) ใน Google Sheets: อัปโหลด CSV แล้วกด File > Download > Microsoft Excel (.xlsx)
  3. 3
    ทำ Mail Merge ใน Word เปิดไฟล์ Template ที่ดาวน์โหลดไว้ ไปที่ Mailings > Start Mail Merge > Letters กด Select Recipients > Use an Existing List แล้วเลือกไฟล์ XLSX
  4. 4
    เลือก Sheet และใส่ Merge Fields เลือก Sheet1$ หรือ sheet ที่มีข้อมูล ติ๊ก First row of data contains column headers ใช้ Insert Merge Field ใส่ช่องข้อมูล เช่น «ฎีกา», «โครงการ», «จำนวน», «บัญชี», «เลขที่»
  5. 5
    Preview และสร้างเอกสารจริง กด Preview Results เพื่อตรวจข้อมูลทีละโครงการ ถ้าถูกต้องแล้วกด Finish & Merge > Edit Individual Documents > All
  6. 6
    Export PDF Word จะสร้างเอกสารใหม่ โดย 1 หน้า = 1 โครงการ กด File > Save As > PDF เพื่อส่งออกไฟล์สุดท้าย
คำสั่งหลักใน Word Mailings > Start Mail Merge > Letters Mailings > Select Recipients > Use an Existing List Mailings > Insert Merge Field Mailings > Preview Results Mailings > Finish & Merge
จุดที่ต้องตรวจซ้ำ แถวแรกของ XLSX ต้องเป็นชื่อคอลัมน์ ห้าม merge cell และควรเก็บข้อมูลไว้ใน sheet เดียว หลังเชื่อมกับ Word แล้ว อย่าย้ายไฟล์ XLSX เพราะ Word จะจำ path ของไฟล์ไว้ Microsoft Support: Mail Merge
Actions เตรียมไฟล์สำหรับ Mail Merge ทำในเว็บหน้านี้ให้ครบก่อน แล้วค่อยไปแปลง XLSX และ merge ใน Word
1. ดาวน์โหลด Word Template ดาวน์โหลด Template ได้ไฟล์ .docx สำหรับใช้เป็น Word Template ของเอกสารปิดโครงการ
2. Export CSV จากระบบ
ค่าเริ่มต้น export ทุกเดือน โดยอ้างอิงกำหนดปิดโครงการ
ได้ไฟล์ .csv ที่มีข้อมูลโครงการ ผู้ตรวจ สถานะปิด งบใช้จริง และข้อมูลบัญชีสำหรับ mail merge

Template เลขรันเอกสาร

คู่มือใช้งาน Template เลขรันเอกสารที่สรุปจากโครงสร้างชีทจริง สำหรับตั้งค่าไฟล์ใหม่และส่งต่อให้เหรัญญิกรุ่นถัดไป

1 ทำสำเนา Template

เปิด Google Sheets Template เลขรันเอกสาร แล้วกด File > Make a copy หรือ ไฟล์ > ทำสำเนา จากนั้นตั้งชื่อไฟล์ตามปีการศึกษา เช่น SGCU68-10 เลขรันเอกสาร

2 ตั้งค่าชีทหน้าแรก

เข้าไปที่ชีท เลขรันเอกสาร แล้วกรอกปีการศึกษาใน B1 และแนบลิงก์ Project Status ของปีนั้นใน B2 โดยต้องเปิดไฟล์ Google Sheets Project Status ต้นทาง กด Share/แชร์ แล้ว Copy link มาใส่

3 ตรวจการดึงข้อมูล

หลังใส่ลิงก์แล้วให้ลองเลือกโครงการในชีทเอกสารพิเศษ หากรหัสโครงการ ชื่อโครงการ หน่วยงาน หรือฝ่ายไม่ขึ้น ให้ตรวจสิทธิ์ IMPORTRANGE และลิงก์ใน B2

ใช้เมื่อต้องลดงบประมาณจากโครงการหนึ่งและเพิ่มงบประมาณให้อีกโครงการหนึ่ง

ต้องกรอก
C:G โครงการลดงบประมาณ, H:L โครงการเพิ่มงบประมาณ, M สถานะ, N หมายเหตุ
อัตโนมัติ
A ลำดับ, B เลขรัน และคอลัมน์ซ่อน O:T
เงื่อนไขเลขรัน
ต้องมีรหัสโครงการอย่างน้อย 1 ช่องใน C:L และมีสถานะใน M
ฎีกา

ใช้กับเอกสารเรียกเงินยืมรองจ่ายคืน/ฎีกา โดยแยกเลขตามประเภทองค์กรและสถานะปิดโครงการ

ต้องกรอก
D รหัสโครงการ และตรวจสถานะใน C/H ตามข้อมูล Project Status
อัตโนมัติ
A ลำดับ, B เลขรันเอกสาร, E ชื่อโครงการ, F หน่วยงาน, G ฝ่าย และคอลัมน์ซ่อน I:S
รูปแบบเลข
SGCU, SCCU, ART, VCK, SPT, PHT หรือ อบจ. 10.01.03-XXX/ปี เมื่อเป็น “ไม่ส่งปิดโครงการ”
TEMPLATE และ SETTING

สองชีทนี้ใช้ดูแลโครงสร้างไฟล์ ไม่ใช่พื้นที่กรอกเอกสารประจำวัน

TEMPLATE
ใช้ copy เพื่อสร้างชีทใหม่ แล้วเปลี่ยนชื่อชีทตามฝ่ายหรือประเภทเอกสารที่ต้องการ
SETTING
ใช้สูตรดึงข้อมูลจาก Project Status เพื่อป้อน dropdown และข้อมูลอัตโนมัติให้ชีทเอกสารพิเศษ ผู้ใช้ทั่วไปไม่ควรแก้
ตัวอย่าง logic ยืมรองจ่าย/ฎีกา
โครงการ A เบิกจริง 5,000 บาท เคยยืมรองจ่าย 4,000 บาท จึงเบิกเพิ่ม 1,000 บาท และสรุปยอดเบิก 5,000 บาท โครงการ B เบิกจริง 3,000 บาท เคยยืมรองจ่าย 4,000 บาท จึงคืนเงิน 1,000 บาท และสรุปยอดเบิก 3,000 บาท

ไฟล์ที่ตรวจพบไม่มีคอลัมน์จำนวนเงินให้กรอกโดยตรงในชีท ฎีกา; หากมีการผูกยอดจาก Project Status เพิ่มเติม ควรตรวจสอบสูตรในชีทจริงอีกครั้งก่อนใช้งาน

ข้อควรระวังและปัญหาที่พบบ่อย
  • ห้ามลบแถวที่เคยออกเลขแล้ว ให้เปลี่ยนสถานะเป็น “ยกเลิก” และใส่เหตุผลในหมายเหตุแทน
  • ห้ามแก้เลขรัน หัวคอลัมน์ หรือสูตรในแถวที่ 4 เพราะหลายชีทใช้สูตรแบบ array
  • ถ้าเลขรันไม่ขึ้น ให้ตรวจช่องบังคับของชีทนั้น ปีการศึกษาใน B1 และประเภทเอกสารที่ C1
  • ถ้าข้อมูลโครงการไม่ขึ้นหรือ dropdown ว่าง ให้ตรวจลิงก์ Project Status ใน B2 และสิทธิ์ IMPORTRANGE
