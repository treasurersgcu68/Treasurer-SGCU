Day 1

1. รับสิทธิ์และรู้จักไฟล์หลัก

ให้รุ่นเดิมเพิ่มบัญชี Google ของเหรัญญิกใหม่เข้า Google Group และแชร์ไฟล์หลักทั้งหมดก่อนเริ่มแก้ข้อมูล

  • เปิด สารบัญ Project Status เพื่อตรวจปีการศึกษาที่ใช้งาน
  • เปิด Project Status ของปีนั้นจากลิงก์ CSV/Google Sheet ในสารบัญ
  • เปิด Template เลขรันเอกสาร เพื่อดูเลขเอกสารที่ออกไปแล้ว
  • เปิดหน้าเว็บจริงแล้วเทียบว่าข้อมูลในเว็บตรงกับชีทหรือไม่
Day 2

2. อ่าน Project Status ให้เป็น

Project Status คือฐานกลางของงานเหรัญญิก ทุกแถวคือหนึ่งโครงการ และรหัสโครงการคือกุญแจที่เชื่อมไปเลขรัน เอกสารยืมรองจ่าย ฎีกา และ dashboard

Routine

3. งานที่ต้องทำเป็นรอบ

แบ่งงานเป็นรอบสั้น ๆ จะไม่หลุดคิวเอกสารและรู้ทันว่าโครงการไหนต้องตามต่อ

  1. ทุกวันดูโครงการที่สถานะเปลี่ยน, งานรอตรวจ, งานใกล้ครบกำหนด และข้อความจากฝ่าย/ชมรม
  2. ทุกสัปดาห์ตรวจยอดงบใช้จริง, งานยืมรองจ่าย, งานรอส่งกิจการนิสิต และงานปิดโครงการค้าง
  3. ทุกเดือนexport รายการปิดโครงการ, ตรวจเลขฎีกา, เทียบเลขรันกับ Project Status และสำรองไฟล์
Handover

4. ก่อนส่งต่อรุ่นถัดไป

รุ่นถัดไปต้องรู้สถานะล่าสุด ไม่ใช่แค่ได้ลิงก์ไฟล์ ให้เตรียมรายการงานค้างและอธิบายจุดเสี่ยงให้ครบ

  • แยกโครงการค้างเป็น รออนุมัติ, รอยืมรองจ่าย, รอปิด, รอฎีกา
  • ส่งรายชื่อผู้รับผิดชอบ ผู้ตรวจ และช่องทางติดต่อที่ยังใช้งานได้
  • บอกไฟล์ไหนเป็นตัวจริง ไฟล์ไหนเป็นสำเนาหรือไฟล์เก่า
  • อธิบายสูตรและ dropdown ที่เคยพังหรือเคยแก้เฉพาะหน้า

    เพิ่มแถวใหม่ใน ตารางสถานะ กรอกรหัส ชื่อ ฝ่าย/ชมรม สถานะอนุมัติ แหล่งเงิน และงบประมาณ

  2. ตามสถานะอนุมัติ

    อัปเดตสถานะในคอลัมน์อนุมัติและโอนงบให้ตรงกับเอกสารจริง ทุกครั้งที่มีมติหรือส่งกิจการนิสิต

  3. จัดการยืมรองจ่าย

    ถ้าโครงการยืมเงิน ให้เปิดเลขรันใน Template แล้วตรวจว่าเลขและยอดยืมกลับมาแสดงใน Project Status

  4. ตามเอกสารปิดโครงการ

    เมื่อจบงาน ให้กรอกงบใช้จริง วันที่ส่งปิด และสถานะปิดโครงการ เพื่อให้ dashboard รู้ว่าค้างหรือเสร็จแล้ว

  5. ออกฎีกา/สรุปเบิกคืน

    ตรวจยอดเบิกเพิ่มหรือคืนเงิน เทียบเลขฎีกา แล้ว export CSV ไปทำเอกสารปิดโครงการถ้าต้องส่งเป็นชุด

ข้อควรระวังที่ต้องย้ำกับรุ่นถัดไป ห้ามลบสูตร ห้ามแก้ชื่อคอลัมน์ และห้าม sort เฉพาะบางคอลัมน์ ให้เลือกทั้งตารางก่อน sort เสมอ ถ้าเพิ่มแถวเกินช่วงเดิม ต้องขยายสูตร, dropdown, filter และช่วง IMPORTRANGE ให้ครอบคลุมแถวใหม่ ถ้าเลขรันเอกสารหรือข้อมูลจากระบบอื่นไม่ขึ้น ให้ตรวจสิทธิ์ IMPORTRANGE, ลิงก์ชีทต้นทาง และรหัสโครงการให้ตรงกันก่อนแก้สูตร

ปิดโครงการและ Export CSV

ใช้เมื่อต้องสรุปโครงการที่ถึงรอบปิด ทำเอกสาร Word หลายโครงการพร้อมกัน และตรวจยอดเบิก/คืนก่อนส่งต่อ

1 แถว = 1 โครงการ แปลง CSV เป็น XLSX ก่อน merge
Source Project Status ต้องกรอกงบใช้จริง วันที่ส่งปิด สถานะปิด และเลขฎีกาให้เรียบร้อยก่อน export
Output CSV/XLSX + Word Template CSV จากเว็บใช้เป็นฐานข้อมูล ส่วน Word Template ใช้สร้างเอกสารปิดโครงการแบบ Mail Merge

ทำตามลำดับนี้เมื่อถึงรอบปิดโครงการ

  1. 1
    กรองโครงการที่ต้องปิด ใน Project Status ดูโครงการที่วันปฏิบัติงานสิ้นสุดแล้วหรือครบกำหนดส่งเอกสาร แยกงานเป็น รอเอกสารจากฝ่าย/ชมรม, รอตรวจ, รอส่งกิจการนิสิต, ส่งเรียบร้อย
  2. 2
    กรอกข้อมูลปิดให้ครบ กรอกงบใช้จริง วันที่ส่งปิด และสถานะปิดโครงการใน Project Status ตรวจยอดเบิกเพิ่ม/คืนเงินจากช่องคำนวณก่อนออกเอกสาร
  3. 3
    ตรวจเลขฎีกา ออกเลขใน Template เลขรันเอกสาร แล้วดูว่าเลขกลับมาแสดงใน Project Status ถ้าเลขไม่ขึ้น ให้ตรวจรหัสโครงการและสิทธิ์ IMPORTRANGE
  4. 4
    Export CSV จากเว็บ เลือกเดือนสุดท้ายที่ต้องการ export หรือปล่อยเป็นทั้งหมด กด Export CSV ปิดโครงการ แล้วเปิดไฟล์เพื่อตรวจหัวคอลัมน์และจำนวนแถว
  5. 5
    แปลง CSV เป็น XLSX ใน Excel: เปิด CSV แล้วกด File > Save As > Excel Workbook (*.xlsx) ใน Google Sheets: อัปโหลด CSV แล้วกด File > Download > Microsoft Excel (.xlsx)
  6. 6
    ทำ Mail Merge ใน Word เปิด Word Template แล้วเลือก Mailings > Start Mail Merge > Letters กด Select Recipients > Use an Existing List แล้วเลือกไฟล์ XLSX Preview ทีละรายการ ก่อน Finish & Merge และ export เป็น PDF
คำสั่งหลักใน Word Mailings > Start Mail Merge > Letters Mailings > Select Recipients > Use an Existing List Mailings > Insert Merge Field Mailings > Preview Results Mailings > Finish & Merge
เช็กก่อนส่งไฟล์ปิดโครงการ ชื่อโครงการ หน่วยงาน เลขฎีกา ยอดเงิน และบัญชีต้องตรงกับ Project Status แถวแรกของ XLSX ต้องเป็นชื่อคอลัมน์ ห้าม merge cell และควรเก็บข้อมูลไว้ใน sheet เดียว หลังเชื่อมกับ Word แล้ว อย่าย้ายไฟล์ XLSX เพราะ Word จะจำ path ของไฟล์ไว้ Microsoft Support: Mail Merge
Actions เครื่องมือที่ใช้ในหน้านี้ ทำข้อมูลใน Project Status ให้ครบก่อน แล้วค่อยดาวน์โหลด template และ export CSV
1. ดาวน์โหลด Word Template ดาวน์โหลด Template ได้ไฟล์ .docx สำหรับใช้เป็น Word Template ของเอกสารปิดโครงการ
2. Export CSV จากระบบ
ค่าเริ่มต้น export ทุกเดือน โดยอ้างอิงกำหนดปิดโครงการ
ได้ไฟล์ .csv ที่มีข้อมูลโครงการ ผู้ตรวจ สถานะปิด งบใช้จริง และข้อมูลบัญชีสำหรับ Mail Merge
3. หลัง export ต้องตรวจ จำนวนแถวใน CSV เท่ากับจำนวนโครงการที่จะออกเอกสาร ไม่มีช่องเลขฎีกา ยอดเงิน หรือบัญชีว่างในรายการที่ต้องส่งจริง ชื่อคอลัมน์ยังอ่านได้ใน Excel/Word ไม่กลายเป็นอักขระเพี้ยน

เลขรันเอกสาร

ระบบเลขรันคือสมุดทะเบียนเอกสารของฝ่ายเหรัญญิก ใช้คุมว่าเอกสารใดออกเลขแล้ว อยู่สถานะไหน และเชื่อมกลับไป Project Status อย่างไร

1 ทำสำเนาไฟล์สำหรับปีใหม่

เปิด Google Sheets Template เลขรันเอกสาร แล้วกด File > Make a copy จากนั้นตั้งชื่อไฟล์ตามปีการศึกษา เช่น SGCU68-10 เลขรันเอกสาร

2 ตั้งค่าหน้าแรก

เข้าไปที่ชีท เลขรันเอกสาร กรอกปีการศึกษาใน B1 และวางลิงก์ Google Sheets Project Status ของปีนั้นใน B2

3 ทดสอบก่อนใช้จริง

ลองเลือกโครงการในชีทเอกสารยืมรองจ่ายหรือฎีกา ถ้าชื่อโครงการ หน่วยงาน หรือฝ่ายไม่ขึ้น ให้ตรวจสิทธิ์ IMPORTRANGE ก่อนออกเลขจริง

เลขรันต้องรักษาลำดับเพื่อให้ตรวจย้อนหลังได้ ห้ามลบเลขที่เคยออกแล้วแม้เอกสารถูกยกเลิก

ห้ามลบ
ถ้าเอกสารยกเลิก ให้เปลี่ยนสถานะเป็น “ยกเลิก” และเขียนเหตุผลในหมายเหตุ
ห้ามแทรก
เพิ่มรายการใหม่ท้ายตาราง เพื่อลดโอกาสสูตรและลำดับเลขคลาดเคลื่อน
ห้ามแก้เลข

ต้องตรงกัน

เอกสารยืมรองจ่าย

ใช้กับโครงการที่ต้องยืมเงินก่อนหรือเบิกเงินรองจ่าย ระบบจะดึงข้อมูลโครงการมาเติมจาก Project Status

กรอกเอง
D รหัสโครงการ, H ผู้รับผิดชอบ, I สถานะ, J หมายเหตุ
อัตโนมัติ
A ลำดับ, B เลขรัน, C STATUS, E รายการ, F ฝ่าย, G หน่วยงาน
เลขไม่ขึ้น
ตรวจว่า D, H, I กรอกครบ และรหัสโครงการมีอยู่ใน Project Status
หลังออกเลข
กลับไปดู Project Status ว่าเลขยืมรองจ่ายแสดงในช่องที่เกี่ยวข้องแล้ว
เอกสารขอเปลี่ยนเพดานงบ

ใช้เมื่อต้องลดงบจากโครงการหนึ่งและเพิ่มงบให้อีกโครงการหนึ่ง

กรอกเอง
C:G โครงการลดงบประมาณ, H:L โครงการเพิ่มงบประมาณ, M สถานะ, N หมายเหตุ
อัตโนมัติ
A ลำดับ, B เลขรัน และคอลัมน์ซ่อน O:T
เงื่อนไข
ต้องมีรหัสโครงการอย่างน้อย 1 ช่องใน C:L และมีสถานะใน M
ฎีกา

ใช้กับเอกสารปิดโครงการ แยกเลขตามประเภทองค์กรและสถานะปิดโครงการ

กรอกเอง
D รหัสโครงการ และตรวจสถานะใน C/H ตามข้อมูล Project Status
อัตโนมัติ
A ลำดับ, B เลขรันเอกสาร, E ชื่อโครงการ, F หน่วยงาน, G ฝ่าย และคอลัมน์ซ่อน I:S
รูปแบบ
SGCU, SCCU, ART, VCK, SPT, PHT หรือ อบจ. 10.01.03-XXX/ปี เมื่อเป็น “ไม่ส่งปิดโครงการ”
หลังออกเลข
ตรวจว่าเลขฎีกากลับไปขึ้นใน Project Status ก่อน export เอกสารปิด
อ่านยอดเบิก/คืนให้ถูก
ถ้าใช้จริง 5,000 บาท และเคยยืม 4,000 บาท ต้องเบิกเพิ่ม 1,000 บาท แต่ยอดใช้จริงของโครงการคือ 5,000 บาท ถ้าใช้จริง 3,000 บาท และเคยยืม 4,000 บาท ต้องคืน 1,000 บาท แต่ยอดใช้จริงของโครงการคือ 3,000 บาท

ให้ยึด Project Status เป็นฐานตรวจยอด เพราะสูตรคำนวณยอดเบิก/คืนอยู่ฝั่ง Project Status ไม่ใช่ทุกชีทในเลขรันจะมีช่องจำนวนเงินให้กรอกโดยตรง

ปัญหาที่พบบ่อยและวิธีไล่ตรวจ
  • เลขรันไม่ขึ้น: ตรวจช่องบังคับของชีทนั้น ปีการศึกษาใน B1 และประเภทเอกสารที่ C1
  • ข้อมูลโครงการไม่ขึ้น: ตรวจลิงก์ Project Status ใน B2, สิทธิ์ IMPORTRANGE และรหัสโครงการ
  • เลขใน Project Status ไม่อัปเดต: ตรวจว่าชีทเลขรันแชร์สิทธิ์ถูกต้องและสูตรดึงข้อมูลยังครอบคลุมแถวใหม่
  • ต้องยกเลิกเอกสาร: เปลี่ยนสถานะเป็น “ยกเลิก” ใส่เหตุผล และไม่ลบแถวเดิม
